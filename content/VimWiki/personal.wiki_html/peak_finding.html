divide and conquer algorithm

1 2 n/2   n
         
  1. look at n/2 elements, worst case complexity theta(n)
  2. if a[n/2]<a[n/2-1], then only look at left half
  3. else if a[n/2]<a[m/2+1] then only look at the right half
  4. else n/2 is peak

what is the complexity

T(n)=T(n/2)+Θ(1) the Θ(1) is the time it take to compare to its neighbors T(n)=T(n/4)+Θ(1)+Θ(1) ... the complexity is Θ(log n), since there is log n terms in the series.

a is a peak iff a >= c, b ,d, e

solution

  • pick middle column j = m/2
  • find global max on column j at (i,j)
  • compare the horizontal adjacent blocks
  • pick the side that are larger than i,j
    • if neither side is larger, then i,j is a peak
  • solve the new problem with the remaining half
  • continue till you have single column left, find the global peak and done!

complexity

T(n,m) where n m are the rows and cols of the 2d array. T(n,m) = T(n,m/2) + Θ(n) <-- the n is here is two find the global peak of the singler reamaining array there is log(m) number of Θ(n), thus: T(n,m) = Θ(n log m)
